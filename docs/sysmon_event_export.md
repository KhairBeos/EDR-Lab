# Sysmon Event XML Export

Use this guide to export the Sysmon Event ID 1 generated by the safe Atomic Red Team `T1059.001` PowerShell test.

## Find the Event

In the Windows VM, open Event Viewer:

```text
Applications and Services Logs
  -> Microsoft
  -> Windows
  -> Sysmon
  -> Operational
```

Filter for Event ID `1`, then look for the event where:

- Image or process name is `powershell.exe`.
- Command line contains the Atomic Red Team marker or `EDR_DEMO_T1059_001`.

PowerShell search option:

```powershell
Get-WinEvent -LogName "Microsoft-Windows-Sysmon/Operational" -MaxEvents 200 |
  Where-Object { $_.Id -eq 1 -and $_.Message -match "EDR_DEMO_T1059_001|Atomic" } |
  Select-Object -First 5 TimeCreated, Id, ProviderName, Message
```

## Export From Event Viewer

1. Select the matching Sysmon Event ID 1.
2. Open the event details.
3. Choose the XML view.
4. Copy the full XML.
5. Save it on the host project as:

```text
samples/sysmon/art_t1059_001_powershell_event.xml
```

You may also save it to another path and pass `--xml-path`.

## Export With PowerShell

Use PowerShell when you want a repeatable export:

```powershell
$event = Get-WinEvent -LogName "Microsoft-Windows-Sysmon/Operational" -MaxEvents 200 |
  Where-Object { $_.Id -eq 1 -and $_.Message -match "EDR_DEMO_T1059_001|Atomic" } |
  Select-Object -First 1

$event.ToXml() | Set-Content -Encoding UTF8 .\art_t1059_001_powershell_event.xml
```

Copy the XML from the VM to the host project folder, then run:

```powershell
python scripts\demo\run_art_sysmon_demo_validation.py --input xml --xml-path .\samples\sysmon\art_t1059_001_powershell_event.xml --engine all --output summary
```

## Troubleshooting

| Symptom | Likely cause | Check |
| --- | --- | --- |
| Sysmon service missing | Sysmon not installed | `Get-Service Sysmon64` |
| Event ID 1 not generated | Config missing Process Create or command did not run | `Get-WinEvent -ListLog "Microsoft-Windows-Sysmon/Operational"` |
| Wrong event selected | Event is not the PowerShell test | Check `Image`, `CommandLine`, and timestamp |
| XML namespace issues | Partial XML copied | Copy the full `<Event xmlns=...>` document |
| Timestamp confusion | VM timezone differs from host | Prefer `UtcTime` and `TimeCreated SystemTime` |
| PowerShell execution policy issues | Atomic test blocked by policy | Use lab-only execution policy override for the safe test |
